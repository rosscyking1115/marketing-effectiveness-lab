# Phase 39 - Product and Contribution Repositioning

## Purpose

Phase 39 resets the project direction from portfolio presentation toward a useful open product. The repository now describes Marketing Effectiveness Lab as a marketing measurement workbench with a production-grade roadmap and contribution path.

## What Changed

- Reframed the README around users, contributors, and product value.
- Replaced showcase-oriented language with product prototype language.
- Added `CONTRIBUTING.md` with contribution principles, development workflow, and production-grade contribution lanes.
- Added `docs/product-roadmap.md` with the mission, near-term product priorities, non-goals, and production path.
- Updated the public homepage copy to position the project as an open measurement workbench.

## Product Direction

The project should now prioritize:

- Useful measurement workflows.
- Safe data onboarding.
- Persistent experiment and model artifacts.
- Production governance and auditability.
- Contribution paths for people who want to improve open marketing effectiveness tooling.

## Boundary

The current app is still a Streamlit prototype. It can be useful for learning, workflow design, and structured analysis, but production use with private business data requires authentication, governed storage, consent controls, audit logs, and deployment hardening.

## Next Step

Phase 40 should add the first real persistence foundation, such as a local artifact registry interface for model runs, CRM experiment briefs, audience exports, readouts, and learning-library records.

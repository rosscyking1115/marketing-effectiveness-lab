Product Roadmap

An open workbench for practical marketing measurement.

The goal is to help teams validate marketing data, estimate channel impact, compare causal evidence, plan budgets, design CRM experiments, and retain reusable learning without pretending the current prototype is already an enterprise system.

Open Interactive App Contribute
Current State Streamlit prototype

Reusable package code, deterministic demo data, documented contracts, and exportable artifacts.

Product Boundary Not confidential-data ready

No authentication, governed storage, tenant isolation, or approval audit log yet.

Next Product Layer Governed persistence

Model runs, CRM briefs, audience files, readouts, and learning records need stable storage.

Near-Term Priorities

Governed Artifact Storage

Persist model runs, uploaded evidence, CRM briefs, audience files, readouts, and learning records with stable IDs, versioning, metadata, and export history.

Safer Data Onboarding

Strengthen connector validation, remediation guidance, and warehouse-friendly contracts for aggregated weekly marketing data.

Experiment Registry

Store planned, launched, completed, and archived experiments, then connect design, audience assignment, calendar, readout, and learning outcomes.

Multi-User Governance

Separate analyst draft work from approved stakeholder decisions with role boundaries, approval states, and audit events.

Deployment Architecture

Move from Streamlit-only prototype toward an API plus production web app, database, background jobs, observability, dependency scanning, and secure config.

Non-Goals For Now

Raw PII in the public demo Production approval claims Vendor replacement claims Pretending demo data is real brand performance

Principle

Useful measurement tools should be transparent about assumptions, data limits, security boundaries, and where human judgment remains necessary.

Built for practical marketing measurement, with production controls treated as product requirements.
